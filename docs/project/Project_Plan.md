# 校园轻集市 — 项目规划

## 一、项目概述

**项目名称：** 校园轻集市（Campus Market）

**项目定位：** 面向校园场景的轻量级二手交易/信息发布平台前端应用

**技术栈：** Vue 3 + TypeScript + Vite + Pinia + Vue Router

---

## 二、页面清单

| 序号 | 页面名称 | 路由路径 | 优先级 | 说明 |
|------|----------|----------|--------|------|
| 1 | 首页/商品列表 | `/` | P0 | 展示所有商品，支持搜索与筛选 |
| 2 | 商品详情 | `/product/:id` | P0 | 查看商品详细信息 |
| 3 | 发布商品 | `/publish` | P0 | 发布新的二手商品 |
| 4 | 用户登录 | `/login` | P0 | 用户登录页面 |
| 5 | 用户注册 | `/register` | P1 | 新用户注册 |
| 6 | 个人中心 | `/user/:id` | P1 | 查看/编辑个人信息 |
| 7 | 我的发布 | `/user/:id/products` | P1 | 管理自己发布的商品 |
| 8 | 我的收藏 | `/user/:id/favorites` | P2 | 查看收藏的商品 |
| 9 | 消息/聊天 | `/messages` | P2 | 买卖双方沟通 |
| 10 | 订单管理 | `/orders` | P2 | 查看交易订单 |

## 五、开发重点（个人判断）

| 优先级 | 模块 | 理由 |
|--------|------|------|
| ⭐⭐⭐ | 商品列表（首页） | 用户进入的第一眼，决定用户体验 |
| ⭐⭐⭐ | 商品发布 | 核心业务流程，没有发布就没有交易 |
| ⭐⭐⭐ | 用户登录 | 权限控制基础，大部分功能依赖登录 |
| ⭐⭐ | 商品详情 | 购买决策的关键页面 |
| ⭐⭐ | API 封装 | 前后端交互的基础，影响所有功能 |
| ⭐ | 收藏/消息 | 提升用户体验，但不是 MVP 必须 |

## 六、技术要点

1. **Composition API：** 使用 `<script setup>` 语法，保持代码简洁
2. **TypeScript 严格模式：** 充分利用类型系统避免运行时错误
3. **Pinia 模块化：** 按业务模块拆分 Store（userStore、productStore 等）
4. **路由懒加载：** 使用动态 import 优化首屏加载速度
5. **组件复用：** 抽取通用组件（ProductCard、LoadingSpinner 等）

## 七、风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 后端 API 未就绪 | 无法联调 | 使用 Mock 数据先行开发 |
| 时间不足 | 功能无法全部完成 | 优先保证 P0 功能完整 |
| AI 代码质量不可控 | 引入 Bug | 所有 AI 代码必须 Review |
| 技术难点卡壳 | 进度阻塞 | 及时求助教师/同学/AI |
